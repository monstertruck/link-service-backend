# Plan: SQLite CRUD with SQLModel

## Why SQLite + SQLModel
- SQLite is embedded — no separate server, single file on disk, perfect for a personal bookmark catalog
- `sqlmodel` (already in requirements.in) merges SQLAlchemy ORM + Pydantic into one class, so the DB model and API schema share a base with minimal duplication
- `sqlmodel` is added to `pyproject.toml` so `uv sync` picks it up

## New Files

### `app/db.py`
- SQLite engine pointed at `links.db` in the project root
- `create_db_tables()` called at app startup via FastAPI lifespan
- `get_session()` — FastAPI dependency yielding a `Session`

### `app/models/__init__.py` — empty init

### `app/models/link.py`
- `LinkRecord` — SQLModel table (`table=True`) with fields:
  - `id: Optional[int]` (primary key, auto-increment)
  - `url: str` (unique index)
  - `title: Optional[str]`
  - `category: LinkCategory`
  - `summary: str`

### `app/crud/__init__.py` — empty init

### `app/crud/links.py`
CRUD functions (all take a `Session`):
| Function | Description |
|---|---|
| `create_link(session, ...)` | Insert a new `LinkRecord`; raises `409` if URL already exists |
| `get_link(session, id)` | Fetch by primary key; returns `None` if not found |
| `get_link_by_url(session, url)` | Fetch by URL |
| `list_links(session, skip, limit)` | Paginated list of all links |
| `delete_link(session, id)` | Delete by primary key; returns `True` if deleted |
